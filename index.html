СЕКЦИЈА 9.6

Tower of Hanoi

Премести n дискови од клин A на клин C користејќи клин B, без да се стави поголем диск врз помал. Класичен пример на рекурзија.

🗼
Рекурзивна идеја: За да се придвижат n дискови A→C:
  1. Премести горните n−1 дискови: A → B (користи C)
  2. Премести најголемиот диск: A → C
  3. Премести n−1 дискови: B → C (користи A)
T(n) = 2·T(n−1) + 1    // рекурентна релација
T(n) = 2ⁿ − 1           // минимален број потези
3
Чекор 0 / 7
Притисни „Напред" за да го следиш чекор по чекор.
Вкупно потези: 7 (= 2ⁿ − 1, n = 3)
⏱ Време: O(2ⁿ) 📦 Простор: O(n) стек
💬 Зошто 2ⁿ − 1? Секое повикување ги удвојува потезите плус 1. T(1)=1, T(2)=3, T(3)=7, T(4)=15... Со 64 дискови би требале 2⁶⁴−1 ≈ 18.4 квинтилиони потези — ако правиш еден во секунда, би траело ~585 милијарди години!
СЕКЦИЈА 9.7

Matrix Chain Multiplication

Наоди го редоследот на множење матрици со минимален вкупен број операции. Заградувањето не го менува резултатот — само цената!

💡
Зошто е важно? За A(10×100)·B(100×5)·C(5×50):
(A·B)·C → 5.000 + 2.500 = 7.500 операции
A·(B·C) → 25.000 + 50.000 = 75.000 операции — 10× поскапо!
cost(i, j) = 0                                // ако i == j
cost(i, j) = mink=i..j-1 { cost(i,k) + cost(k+1,j) + P[i-1]·P[k]·P[j] }
DP ТАБЕЛА — cost(i,j)
Оптимален
Избран
0 (дијагонала)

← Кликни ќелија за детали

Избери ќелија за да видиш кои матрици се множат, сите можни k вредности и оптималното заградување.

⏱ Време: O(n³) 📦 Простор: O(n²)
💬 Клучна идеја: Пополнувај DP по должина на подниз — прво cost(i,i)=0 (должина 1), па должина 2, 3… до n. Секоја ќелија зависи само од помали подниза кои веќе се пресметани.
СЕКЦИЈА 9.8

Lowest Common Ancestor (LCA)

За два јазли u и v во стебло, LCA е најдлабокиот предок кој е заеднички за двата. Кликни на два јазли за да го пронајдеш!

🌳
Наивен пристап: За секој јазол оди нагоре кон коренот, зачувај го патот, пронајди го првиот заеднички јазол → O(n) по прашање.
Binary Lifting: Зачувај предок на 2ᵏ ниво за секој јазол → O(log n) по прашање, O(n log n) предобработка.

АЛГОРИТАМ: НАИВЕН LCA
1
DFS — запиши длабочина и родител
За секој јазол зачувај го родителот и длабочината. Еден DFS = O(n).
2
Изедначи ги длабочините
Поместувај го подлабокиот јазол нагоре додека и двата не бидат на иста длабочина.
3
Движи ги заедно нагоре
Поместувај ги и двата паралелно нагоре — кога ќе се сретнат, тоа е LCA!
★
Binary Lifting (оптимизација)
ancestor[v][k] = ancestor[ancestor[v][k−1]][k−1] — скокај по 2ᵏ чекори наеднаш.
ancestor[v][0] = parent(v)                 // директен родител
ancestor[v][k] = ancestor[ancestor[v][k-1]][k-1]   // 2ᵏ-ти предок
⏱ Наивно: O(n) ⏱ Binary Lift: O(log n) 📦 O(n log n)
